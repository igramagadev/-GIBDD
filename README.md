# Discord Bot - Система Управления Фракцией

Комплексный Discord-бот для автоматизации внутренних процессов во фракции (ГИБДД/Полиция/и т.д.). Бот полностью берет на себя рутину по приему заявок на стажировку, обработке рапортов на увольнение и ведению кадрового аудита (повышение, понижение, увольнение сотрудников).

## Основной функционал

1. Система заявок на стажировку
   - Автоматическое создание панели с кнопкой подачи заявки
   - Модальные окна (формы) для заполнения информации кандидатом
   - Логирование заявок в закрытый канал для проверки руководящим составом
   - Кнопки Одобрить/Отклонить с запросом причины
   - Автоматическая выдача базовой роли и роли курсанта при одобрении

2. Система заявлений на увольнение
   - Отдельная панель для сотрудников, желающих покинуть ряды
   - Указание причины ухода через удобную форму
   - Снятие всех фракционных ролей (званий, отделов) при одобрении
   - Автоматическая выдача роли "Уволен"

3. Кадровый аудит
   - Централизованная панель управления личным составом для старшего состава
   - Принятие на службу (с ручным вводом звания, номера и причины)
   - Повышение и понижение в звании
   - Принудительное увольнение сотрудников
   - Подробное логирование всех кадровых действий в специальный аудит-канал

4. Дополнительные возможности
   - Управление ролями осуществляется на основе удобного конфигурационного файла `.env`
   - Использование современных интерактивных компонентов Discord (Components V2): Модалки и Кнопки без использования устаревших текстовых команд
   - Локальная база данных SQLite для сохранения статусов заявок и пользователей

## Требования

- Python: 3.11
- Зависимости (библиотеки):
  - `disnake` (версия >= 2.12.0)
  - `python-dotenv` (версия >= 1.0.0)
- Операционная система: Windows, Linux или macOS (рекомендуется запуск через Docker для Linux серверов)

## Установка и Запуск

Вы можете запустить бота классическим способом напрямую через Python, либо изолированно с помощью Docker.

### Подготовка (Обязательно для обоих способов)

1. Клонируйте репозиторий или распакуйте архив с ботом в нужную директорию.
2. Переименуйте или создайте файл `.env` в корневой папке проекта.
3. Откройте `.env` и заполните все необходимые ID (токен бота, каналы для панелей и логов, ID ролей, а также списки рангов и отделов). Обратите внимание: формат `RANKS_MAP` должен быть строго вида `Звание:ID`.

---

### Способ 1: Нативный запуск (через Python)

1. Установите строго Python 3.11, если он еще не установлен.
2. Откройте терминал (командную строку) в папке с ботом.
3. Рекомендуется создать виртуальное окружение:
   ```bash
   python -m venv venv
   ```
4. Активируйте виртуальное окружение:
   - Windows: `venv\Scripts\activate`
   - Linux/macOS: `source venv/bin/activate`
5. Установите все зависимости из файла конфигурации:
   ```bash
   pip install -r requirements.txt
   ```
6. Запустите бота:
   ```bash
   python main.py
   ```
   (На некоторых системах команда может быть `python3 main.py` или `py main.py`)

---

### Способ 2: Запуск через Docker

Если на вашем сервере установлен Docker, это самый быстрый и безопасный способ запуска.

1. Убедитесь, что Docker установлен и запущен на вашей системе.
2. Откройте терминал в папке с ботом (где находится файл `Dockerfile`).
3. Соберите образ бота. Вы можете назвать его, например, `discord-bot`:
   ```bash
   docker build -t discord-bot .
   ```
4. Запустите контейнер в фоновом режиме:
   ```bash
   docker run -d --name my-discord-bot discord-bot
   ```

Для просмотра логов работы бота в Docker используйте команду:
```bash
docker logs -f my-discord-bot
```

Чтобы остановить бота:
```bash
docker stop my-discord-bot
```

## Структура проекта

- `main.py` — Точка входа, запуск бота и инициализация модулей.
- `config/settings.py` — Загрузка и валидация конфигурации из `.env`.
- `cogs/applications.py` — Модуль обработки заявок на стажировку и увольнение.
- `cogs/audit.py` — Модуль кадрового аудита (повышения, увольнения, логирование).
- `utils/` — Вспомогательные скрипты: работа с БД (`database.py`), проверки ролей (`helpers.py`), создание UI панелей (`panel_init.py`).
- `disabled_features.py` — Архивный файл с отключенным функционалом (ЧС, переводы, слеш-команды).
- `data/` — Папка, которая создается автоматически и хранит файл базы данных `bot.db`.
